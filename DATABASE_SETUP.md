# Настройка базы данных

Бот поддерживает два варианта базы данных:

## 1. SQLite (локальная база данных) - Рекомендуется для начала

SQLite хранит все данные в одном файле на вашем компьютере. Не требует установки отдельного сервера.

### Настройка:

В файле `config.py` добавьте или раскомментируйте:

```python
DB_PATH = "bot_database.db"  # Файл будет создан автоматически
```

Или закомментируйте `MYSQL_URL`:

```python
# MYSQL_URL = "mysql+mysqlconnector://..."
DB_PATH = "bot_database.db"
```

Файл базы данных `bot_database.db` будет создан автоматически в корне проекта при первом запуске.

**Преимущества:**
- Не требует установки MySQL
- Все данные локально
- Простое резервное копирование (просто скопировать файл)
- Работает "из коробки"

**Недостатки:**
- Менее производительна для очень больших объемов данных
- Не подходит для распределенных систем

## 2. MySQL (удаленный или локальный сервер)

Для использования MySQL настройте подключение в `config.py`:

```python
MYSQL_URL = "mysql+mysqlconnector://user:password@host:port/database_name"
```

**Преимущества:**
- Высокая производительность
- Подходит для больших объемов данных
- Можно использовать на удаленном сервере

**Недостатки:**
- Требует установки и настройки MySQL
- Нужен доступ к серверу

## Переключение между базами данных

1. **Для использования SQLite:** Укажите `DB_PATH` в `config.py`
2. **Для использования MySQL:** Укажите `MYSQL_URL` и не указывайте `DB_PATH` (или установите `DB_PATH = None`)

**Важно:** При переключении между базами данных данные не переносятся автоматически. Нужно будет заново загрузить документы в базу знаний.

## Резервное копирование

### SQLite:
Просто скопируйте файл `bot_database.db`:
```bash
cp bot_database.db bot_database_backup.db
```

### MySQL:
Используйте стандартные инструменты MySQL:
```bash
mysqldump -u user -p database_name > backup.sql
```
